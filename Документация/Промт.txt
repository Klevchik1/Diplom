📊 МОДЕЛИ ДАННЫХ (ПОЛНЫЙ СПИСОК НА ОСНОВЕ models.py):

👤 ПОЛЬЗОВАТЕЛИ И АУТЕНТИФИКАЦИЯ:

User (наследуется от AbstractUser) - Кастомная модель пользователя с email-аутентификацией

Поля: email, name, surname, number
Telegram интеграция: telegram_chat_id, telegram_username, is_telegram_verified, telegram_verification_code
Email верификация: is_email_verified, email_verification_code, email_verification_code_sent_at
Методы: unlink_telegram(), generate_verification_code(), generate_email_verification_code(), is_verification_code_expired(), verify_email(), requires_email_verification()
Кастомный менеджер: CustomUserManager

PendingRegistration - Временные данные для верификации email при регистрации

Поля: email, name, surname, number, password, verification_code, created_at
Методы: is_expired(), create_user()
Автоматическое удаление через 30 минут

PasswordResetRequest - Запросы на сброс пароля

Поля: user, reset_code, created_at, expires_at, is_used
Методы: is_expired(), mark_as_used()
30-минутная валидность кодов

EmailChangeRequest - Запросы на смену email

Поля: user, new_email, verification_code, created_at, expires_at, is_used
Методы: is_expired(), mark_as_used()
30-минутная валидность

🎬 КИНОТЕАТРАЛЬНАЯ ЛОГИКА:

Country - Страны

Поля: name, code, created_at
Уникальные коды стран (RU, US и т.д.)

HallType - Типы залов с коэффициентами цен

Поля: name, description, price_coefficient, base_price, created_at
Определяет базовую стоимость для залов данного типа

Hall - Кинозал со схемой мест

Поля: name, rows, seats_per_row, description, hall_type, created_at, updated_at
Связи: hall_type (ForeignKey)
Методы: delete(), save(), create_seats()
Автоматическое создание мест при сохранении

Director - Режиссёры

Поля: name, surname, birth_date, country, biography, created_at
Связи: country (ForeignKey)

Actor - Актёры

Поля: name, surname, birth_date, country, biography, created_at
Связи: country (ForeignKey)

Genre - Жанры фильмов

Поля: name, description, created_at
Уникальные названия с автоформатированием
AgeRating - Возрастные рейтинги
Поля: name (0+, 6+, 12+, 16+, 18+), created_at

Movie - Информация о фильме

Поля: title, short_description, description, duration (IntegerField), poster, release_year, genre, age_rating, created_at
Связи: genre (ForeignKey), age_rating (ForeignKey), directors (ManyToMany через MovieDirector), actors (ManyToMany через MovieActor)
Методы: get_duration_display(), save()
Автоматическое создание короткого описания из полного

MovieDirector - Связующая модель для фильмов и режиссёров

Поля: movie, director, created_at
unique_together: ('movie', 'director')

MovieActor - Связующая модель для фильмов и актёров

Поля: movie, actor, created_at
unique_together: ('movie', 'actor')

Screening - Сеанс (привязывает фильм к залу и времени)

Поля: movie, hall, start_time, end_time, ticket_price, created_at
Связи: movie (ForeignKey), hall (ForeignKey)
Методы: clean(), calculate_ticket_price(), get_time_multiplier(), get_price_calculation_explanation(), get_time_description(), save()
Автоматический расчет цены на основе:
Типа зала (base_price * price_coefficient)
Времени суток (утро 0.7, день 0.9, вечер 1.2, ночь 1.4)
Проверка пересечений сеансов
Проверка рабочего времени (8:00-23:00, окончание до 00:30)

Seat - Место в зале

Поля: hall, row, number, created_at
Связи: hall (ForeignKey)
unique_together: ('hall', 'row', 'number')

TicketStatus - Статусы билетов

Поля: code, name, description, is_active, can_be_refunded, created_at
Основные статусы: active, refunded, refund_requested
Управляется через админку

TicketGroup - Группа билетов одной покупки

Поля: group_uuid, user, screening, purchase_date, total_amount, tickets_count, created_at
Связи: user (ForeignKey), screening (ForeignKey)
UUID для удобного скачивания группы

Ticket - Билет

Поля: user, screening, seat, qr_code, price, refund_requested_at, refund_processed_at, status, ticket_group, created_at, updated_at
Связи: user (ForeignKey), screening (ForeignKey), seat (ForeignKey), status (ForeignKey), ticket_group (ForeignKey)
unique_together: ('screening', 'seat')
Методы: can_be_refunded(), request_refund(), process_refund(), cancel_refund_request(), get_status_display(), get_pdf_url(), get_group_tickets()
Автоматическая система возврата с проверкой:
Минимум 30 минут до начала сеанса
Статус билета
Автоматическая обработка возвратов

🛠️ СИСТЕМНЫЕ МОДЕЛИ:

ActionType - Типы действий для логирования

Поля: code, name, description, created_at
Коды: CREATE, UPDATE, DELETE, VIEW, EXPORT, LOGIN, LOGOUT, BACKUP, REPORT, OTHER

ModuleType - Типы модулей для логирования

Поля: code, name, description, created_at
Коды: USERS, MOVIES, HALLS, SCREENINGS, TICKETS, REPORTS, BACKUPS, SYSTEM, AUTH

OperationLog - Логирование действий в системе

Поля: user, action_type, module_type, description, ip_address, user_agent, object_id, object_repr, additional_data (JSONField), timestamp
Связи: user (ForeignKey), action_type (ForeignKey), module_type (ForeignKey)
Методы: get_additional_data_display()

BackupManager - Управление бэкапами БД

Поля: name, backup_file, backup_type (full/daily), backup_date, restored_at, restoration_status (pending/in_progress/completed/failed), restoration_log, user, created_at
Связи: user (ForeignKey)
Методы: get_file_path(), file_exists(), file_size(), can_be_restored(), restore_database(), get_restoration_status_display(), get_restoration_color(), get_download_url(), get_absolute_path()
Типы: full, daily

Report - Модель-заглушка для отображения отчетов в админке
